## Overview

The Users API provides endpoints for managing system users, staff members, and crew members in SIGSAHO. It supports user creation, retrieval, updates, role assignment, and soft deletion.

## Authentication

All endpoints require authentication via JWT cookie (`cookieAuth`).

## Base URL

```
/api/users
```

---

## Create User

Create a new system user with specified roles and login access.

<CodeGroup>
```bash POST /users
curl -X POST https://api.sigsaho.com/api/users \
  -H "Content-Type: application/json" \
  -b "cookies.txt" \
  -d '{
    "name": "Juan",
    "last_name": "García",
    "second_last_name": "López",
    "username": "jgarcia",
    "email": "juan.garcia@example.com",
    "phone": "+525512345678",
    "password": "SecurePass123!",
    "weight": 75,
    "roles": [1, 3],
    "login_access": true,
    "birth_date": "1990-05-15T00:00:00.000Z",
    "position": "Piloto Comercial",
    "license_number": "PIL-2024-001"
  }'
```
</CodeGroup>

### Request Body

<ParamField body="name" type="string" required>
  User's first name (1-100 characters)
</ParamField>

<ParamField body="last_name" type="string" required>
  User's paternal last name (1-100 characters)
</ParamField>

<ParamField body="second_last_name" type="string">
  User's maternal last name (max 100 characters)
</ParamField>

<ParamField body="username" type="string" required>
  Unique username (3-50 characters, alphanumeric with dots, hyphens, underscores)
</ParamField>

<ParamField body="email" type="string" required>
  Unique email address (max 255 characters)
</ParamField>

<ParamField body="phone" type="string">
  Phone number (10-15 digits, international format supported)
</ParamField>

<ParamField body="password" type="string">
  Password (min 8 characters, must include uppercase, lowercase, number, and special character). **Required when `login_access` is `true`**
</ParamField>

<ParamField body="weight" type="number" required>
  User's weight in kg (positive number, max 500)
</ParamField>

<ParamField body="roles" type="array" required>
  Array of role IDs to assign (min 1, max 10)
</ParamField>

<ParamField body="login_access" type="boolean" required>
  Whether the user can log into the system
</ParamField>

<ParamField body="birth_date" type="string">
  Date of birth (ISO 8601 format)
</ParamField>

<ParamField body="position" type="string">
  User's position/job title (max 100 characters)
</ParamField>

<ParamField body="license_number" type="string">
  License number (max 50 characters)
</ParamField>

<ResponseField name="user" type="object">

    </ResponseField>
    <ResponseField name="last_name" type="string">

    </ResponseField>
    <ResponseField name="second_last_name" type="string">
      Maternal last name
    </ResponseField>
    <ResponseField name="username" type="string">
      Username
    </ResponseField>
    <ResponseField name="email" type="string">
      Email address
    </ResponseField>
    <ResponseField name="phone" type="string">
      Phone number
    </ResponseField>
    <ResponseField name="login_access" type="boolean">
      Login access status
    </ResponseField>
    <ResponseField name="weight" type="number">
      Weight in kg
    </ResponseField>
    <ResponseField name="license_number" type="string">
      License number
    </ResponseField>
    <ResponseField name="birth_date" type="string">
      Date of birth
    </ResponseField>
    <ResponseField name="position" type="string">

## Create Staff User

<CodeGroup>
```bash POST /users/staff

  -b "cookies.txt" \
  -F "name=María" \
  -F "last_name=Rodríguez" \
  -F "second_last_name=Pérez" \
  -F "email=maria.rodriguez@example.com" \
  -F "phone=+525587654321" \
  -F "weight=65" \
  -F "position=Enfermera" \
  -F "license_file=@/path/to/license.pdf" \
  -F "license_serial_number=LIC-2024-123" \
```

## List Users

Retrieve a paginated list of users with optional filters.

<CodeGroup>
```bash GET /users
curl -X GET "https://api.sigsaho.com/api/users?page=1&limit=10&status=ACTIVE&search=juan" \
  -b "cookies.txt"
```
</CodeGroup>

### Query Parameters

<ParamField query="page" type="integer" default="1">
  Page number (min: 1)
</ParamField>

<ParamField query="limit" type="integer" default="10">
  Results per page (min: 1, max: 100)
</ParamField>

<ParamField query="status" type="string">
  Filter by status: `ACTIVE`, `DISABLED`, or `ELIMINATED`
</ParamField>

<ParamField query="login_access" type="boolean">
  Filter by login access (true/false)
</ParamField>

<ParamField query="search" type="string">
  Search by name, last name, username, or email
</ParamField>

### Response

<ResponseField name="data" type="array">
  Array of user objects
</ResponseField>

<ResponseField name="pagination" type="object">
  <Expandable title="Pagination info">
    <ResponseField name="total" type="number">
      Total number of users
    </ResponseField>
    <ResponseField name="page" type="number">
      Current page
    </ResponseField>
    <ResponseField name="limit" type="number">
      Results per page
    </ResponseField>
    <ResponseField name="totalPages" type="number">
      Total number of pages
    </ResponseField>
  </Expandable>
</ResponseField>

### Authorization

**Required Role:** `ADMINISTRADOR` or `DIRECTOR`

## Reset Password

Reset a user's password to a new value or generate a temporary password.

<CodeGroup>
```bash POST /users/:id/reset-password
curl -X POST https://api.sigsaho.com/api/users/42/reset-password \
  -H "Content-Type: application/json" \
  -b "cookies.txt" \
  -d '{"password": "NewSecurePass123!"}'
```

```bash Generate Temporary Password
curl -X POST https://api.sigsaho.com/api/users/42/reset-password \
  -H "Content-Type: application/json" \
  -b "cookies.txt" \
  -d '{}'
```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="integer" required>
  User ID
</ParamField>

### Request Body

<ParamField body="password" type="string">
  New password (min 8 characters with complexity requirements). If omitted, a temporary password is generated.
</ParamField>

### Response

<ResponseField name="message" type="string">
  Success message
</ResponseField>

<ResponseField name="temporaryPassword" type="string">
  Generated temporary password (only present if password was not provided)
</ResponseField>

<ResponseField name="user" type="object">
  Updated user object
</ResponseField>

<ResponseExample>
```json 200 Success (Generated Password)
{
  "message": "Contraseña restablecida exitosamente",
  "temporaryPassword": "Abc123456@1x",
  "user": {
    "id": 42,
    "name": "Juan",
    "username": "jgarcia"
  }
}
```

```json 429 Rate Limit
{
  "error": "Demasiadas solicitudes - Rate limit crítico excedido (3/hora)"
}
```
</ResponseExample>

### Rate Limiting

**Critical operation:** Limited to **3 requests per hour per IP**

### Authorization

**Required Role:** `ADMINISTRADOR`

**Note:** Cannot reset password for admin user (ID 1)

## User Types

### SYSTEM
Users with login access to the system. Can have various roles and permissions.

### STAFF
General staff members without login access. Used for identification and document management.

---

## User Status Values

- **ACTIVE**: User is active and operational
- **DISABLED**: User is temporarily disabled
- **ELIMINATED**: User has been soft-deleted

---

## Error Codes

| Code | Description |
|------|-------------|
| 400  | Bad request - validation error |
| 401  | Unauthorized - authentication required |
| 403  | Forbidden - insufficient permissions |
| 404  | User not found |
| 409  | Conflict - duplicate email or username |
| 429  | Rate limit exceeded |
| 500  | Internal server error |
