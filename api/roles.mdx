## Overview

The Roles API provides endpoints for retrieving system roles. Roles define permissions and access levels for users in SIGSAHO.

## Authentication

All endpoints require authentication via JWT cookie (`cookieAuth`).

## Base URL

```
/api/roles
```

---

## Get Role by ID

Retrieve a specific role by its ID.

<CodeGroup>
```bash GET /roles/:id
curl -X GET https://api.sigsaho.com/api/roles/1 \
  -b "cookies.txt"
```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="integer" required>
  Role ID
</ParamField>

### Response

<ResponseField name="role" type="object">
  Role object with details
  
  <Expandable title="Role properties">
    <ResponseField name="id" type="number">
      Role ID
    </ResponseField>
    <ResponseField name="name" type="string">
      Role name
    </ResponseField>
    <ResponseField name="created_at" type="string">
      Creation timestamp
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>
```json 200 Success
{
  "role": {
    "id": 1,
    "name": "ADMINISTRADOR",
    "created_at": "2024-01-01T00:00:00.000Z"

```

```json 400 Invalid ID
{
  "error": "ID inválido"
}
```

```json 404 Not Found
{
  "error": "Rol no encontrado"
}
```
</ResponseExample>

### Authorization

**Required:** Valid JWT authentication (any authenticated user)

## System Roles

SIGSAHO defines the following system roles with distinct permissions:

### Administrative Roles

<AccordionGroup>
  <Accordion title="ADMINISTRADOR" icon="user-shield">
    **Full system access**
    
    - Complete control over all modules
    - User and role management
    - System configuration
    - Access to all data and operations
    - Can create, read, update, and delete all resources
    
    **Login Access:** Yes
  </Accordion>
  
  <Accordion title="DIRECTOR" icon="eye">
    **Read-only access to all modules**
    
    - View-only permissions across the system
    - Cannot modify, create, or delete resources
    - Access to all reports and analytics
    - Ideal for supervisory oversight
    
    **Login Access:** Yes
  </Accordion>
</AccordionGroup>

### Operational Roles

<AccordionGroup>
  <Accordion title="EDITOR-TM" icon="helicopter">
    **Medical Transfer Editor**
    
    - Edit and manage medical air transfers (Traslados Aeromédicos)
    - View patient information
    - Update transfer status and details
    - Cannot access general transfers
    
    **Login Access:** Yes
  </Accordion>
  
  <Accordion title="EDITOR-TG" icon="plane">
    **General Transfer Editor**
    
    - Edit general transfers (Security, Civil Protection, Maintenance)
    - Manage non-medical flight operations
    - Cannot access medical transfers
    
    **Login Access:** Yes
  </Accordion>
  
  <Accordion title="MANTENIMIENTO" icon="wrench">
    **Maintenance Manager**
    
    - Manage aircraft maintenance operations
    - Schedule and track aircraft services
    - Manage parts inventory
    - Configure maintenance notifications
    
    **Login Access:** Yes
  </Accordion>
  
  <Accordion title="CAPACITACION" icon="graduation-cap">
    **Training Manager**
    
    - Manage staff training programs
    - Track certifications and qualifications
    - Schedule training sessions
    - Monitor training compliance
    
    **Login Access:** Yes
  </Accordion>
  
  <Accordion title="DESPACHADOR" icon="tower-broadcast">
    **Air Traffic Dispatcher**
    
    - Flight dispatch and control operations
    - Manage flight plans
    - Coordinate crew assignments
    - Monitor active flights
    
    **Login Access:** Yes
  </Accordion>
</AccordionGroup>

### Crew Identification Roles

These roles are used for identification purposes only and do not have login access.

<AccordionGroup>
  <Accordion title="MEDICO" icon="user-doctor">
    **Medical Doctor**
    
    - Identification role for medical personnel
    - Used in crew member assignments
    - No system login access
    - Tracked for flight operations and compliance
    
    **Login Access:** No
  </Accordion>
  
  <Accordion title="PILOTO" icon="user-pilot">
    **Pilot**
    
    - Identification role for pilots
    - Used in crew member assignments
    - No system login access
    - Tracked for flight hours and certifications
    
    **Login Access:** No
  </Accordion>
  
  <Accordion title="ENFERMERO" icon="user-nurse">
    **Nurse**
    
    - Identification role for nursing personnel
    - Used in crew member assignments
    - No system login access
    - Tracked for flight operations
    
    **Login Access:** No
  </Accordion>
  
  <Accordion title="TSUP" icon="user-gear">
    **TSUP (Technical Support)**
    
    - Identification role for technical support personnel
    - Used in crew member assignments
    - No system login access
    - Tracked for specialized operations
    
    **Login Access:** No
  </Accordion>
</AccordionGroup>

## Role Assignment

Roles are assigned to users through the [Users API](/api/users). A user can have multiple roles assigned simultaneously.

### Assignment Rules

1. **Minimum Roles:** Every user must have at least one role
2. **Maximum Roles:** A user can have up to 10 roles
3. **Role Validation:** Only valid role IDs can be assigned
4. **Login Access:** Users with crew identification roles (MEDICO, PILOTO, ENFERMERO, TSUP) should have `login_access` set to `false`

### Example Assignment

```json
{
  "name": "Juan",
  "last_name": "García",
  "email": "juan.garcia@example.com",
  "roles": [1, 9],  // ADMINISTRADOR + PILOTO
  "login_access": true
}
```

## Permission Matrix

| Module | ADMIN | DIRECTOR | EDITOR-TM | EDITOR-TG | MANTENIMIENTO | CAPACITACION | DESPACHADOR |
|--------|-------|----------|-----------|-----------|---------------|--------------|-------------|
| Users | CRUD | Read | - | - | - | - | - |
| Roles | Read | Read | - | - | - | - | - |
| Medical Transfers | CRUD | Read | CRUD | - | - | - | Read |
| General Transfers | CRUD | Read | - | CRUD | - | - | Read |
| Aircraft Maintenance | CRUD | Read | - | - | CRUD | - | Read |
| Training | CRUD | Read | - | - | - | CRUD | - |
| Flight Dispatch | CRUD | Read | Read | Read | Read | - | CRUD |
| Reports | CRUD | Read | Read | Read | Read | Read | Read |

**Legend:**
- CRUD: Create, Read, Update, Delete
- Read: Read-only access
- `-`: No access
