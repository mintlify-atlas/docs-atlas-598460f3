## Overview

The Transfer Statistics API provides comprehensive analytics and reporting endpoints for transfer operations. All statistics exclude canceled transfers by default and support date range filtering.

All endpoints require JWT authentication via cookies and the `stats:read` permission.

## General Statistics

```http
GET /transfer-statistics/stats/general
```

Returns total transfers and breakdown by type (MEDICAL, CIVIL_DEFENSE, SECURITY). Excludes MAINTENANCE, OTHER types and CANCELED transfers.

### Authentication
Requires JWT authentication and `stats:read` permission.

### Query Parameters

<ParamField query="date_from" type="string">
Filter from date in YYYY-MM-DD format
</ParamField>

<ParamField query="date_to" type="string">
Filter to date in YYYY-MM-DD format
</ParamField>

### Response

<ResponseField name="total" type="integer">
Total number of transfers
</ResponseField>

<ResponseField name="byType" type="object">
Breakdown by transfer type
</ResponseField>

<ResponseField name="byType.medical" type="integer">
Number of medical transfers
</ResponseField>

<ResponseField name="byType.civilDefense" type="integer">
Number of civil defense transfers
</ResponseField>

<ResponseField name="byType.segurity" type="integer">
Number of security transfers
</ResponseField>

<ResponseField name="filters" type="object">
Applied filters
</ResponseField>

<ResponseField name="filters.dateFrom" type="string">
Start date filter applied (nullable)
</ResponseField>

<ResponseField name="filters.dateTo" type="string">
End date filter applied (nullable)
</ResponseField>

### Example Request

```bash
curl -X GET "https://api.sigsaho.gob.mx/transfer-statistics/stats/general?date_from=2024-01-01&date_to=2024-12-31" \
  -H "Cookie: token=your_jwt_token"
```

### Example Response

```json
{
  "total": 150,
  "byType": {
    "medical": 80,
    "civilDefense": 45,
    "segurity": 25
  },
  "filters": {
    "dateFrom": "2024-01-01",
    "dateTo": "2024-12-31"
  }
}
```

## Monthly Statistics

```http
GET /transfer-statistics/stats/monthly
```

Returns transfer counts grouped by year and month. Includes data for the requested year and 2 previous years. Excludes MAINTENANCE type and CANCELED transfers.

### Authentication
Requires JWT authentication and `stats:read` permission.

### Query Parameters

<ParamField query="year" type="integer">
Base year for query (default: current year). Must be between 2000-2100.
</ParamField>

<ParamField query="status" type="enum">
Filter by specific status. Options: `PROCESS`, `FINISHED`, `CANCELED`
</ParamField>

### Response

<ResponseField name="data" type="array">
Array of monthly statistics objects
</ResponseField>

<ResponseField name="data[].year" type="integer">
Year
</ResponseField>

<ResponseField name="data[].month" type="integer">
Month (1-12)
</ResponseField>

<ResponseField name="data[].count" type="integer">
Number of transfers
</ResponseField>

<ResponseField name="years" type="array">
Array of years included in the response
</ResponseField>

### Example Request

```bash
curl -X GET "https://api.sigsaho.gob.mx/transfer-statistics/stats/monthly?year=2024&status=FINISHED" \
  -H "Cookie: token=your_jwt_token"
```

### Example Response

```json
{
  "data": [
    {
      "year": 2024,
      "month": 1,
      "count": 15
    },
    {
      "year": 2024,
      "month": 2,
      "count": 20
    }
  ],
  "years": [2022, 2023, 2024]
}
```
