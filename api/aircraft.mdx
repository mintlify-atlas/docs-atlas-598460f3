## Create Aircraft

<Note>
  Requires authentication with `requireDespachoWrite` permission
</Note>

Create a new aircraft with an image upload.

<CodeGroup>
```bash cURL
curl -X POST https://api.sigsaho.com/aircrafts \
  -H "Cookie: token=YOUR_JWT_TOKEN" \
  -F "name=Bell 407" \
  -F "registration=XA-ABC" \
  -F "description=Bell 407 helicopter for executive transport" \
  -F "aircraft_category=ROTARY_WING" \
  -F "number_of_engines=1" \
  -F "manufacturer=Bell Helicopter" \
  -F "model=407" \
  -F "time_limit_m1=100" \
  -F "cicle_limit_m1=500" \
  -F "weight_empty=2700" \
  -F "cg_empty=150.5" \
  -F "moment_empty=406350" \
  -F "max_passengers=6" \
  -F "current_time_flying=0" \
  -F "current_cycles=0" \
  -F "total_lifetime_hours=5000" \
  -F "hours_before_lastMantt=0" \
  -F "total_lifetime_cycles=0" \
  -F "image=@aircraft.jpg"
```
</CodeGroup>

### Request

<ParamField body="name" type="string" required>
  Aircraft name
</ParamField>

<ParamField body="registration" type="string" required>
  Aircraft registration number (e.g., XA-ABC)
</ParamField>

<ParamField body="image" type="file" required>
  Aircraft image (JPEG, PNG, or WebP). Maximum 10MB. Image is automatically optimized to 85% quality.
</ParamField>

<ParamField body="description" type="string" required>
  Aircraft description
</ParamField>

<ParamField body="aircraft_category" type="enum" required>
  Aircraft category: `FIXED_WING` or `ROTARY_WING`
</ParamField>

<ParamField body="number_of_engines" type="integer" required>
  Number of engines (default: 1)
</ParamField>

<ParamField body="manufacturer" type="string">
  Aircraft manufacturer
</ParamField>

<ParamField body="model" type="string">
  Aircraft model
</ParamField>

<ParamField body="time_limit_m1" type="number" required>
  Time limit for engine 1 (hours)
</ParamField>

<ParamField body="time_limit_m2" type="number">
  Time limit for engine 2 (hours, if applicable)
</ParamField>

<ParamField body="cicle_limit_m1" type="integer" required>
  Cycle limit for engine 1
</ParamField>

<ParamField body="cicle_limit_m2" type="integer">
  Cycle limit for engine 2 (if applicable)
</ParamField>

<ParamField body="weight_empty" type="number" required>
  Empty weight (kg)
</ParamField>

<ParamField body="cg_empty" type="number" required>
  Center of gravity when empty
</ParamField>

<ParamField body="moment_empty" type="number" required>
  Moment when empty
</ParamField>

<ParamField body="max_passengers" type="integer" required>
  Maximum number of passengers
</ParamField>

<ParamField body="current_time_flying" type="number">
  Current flight hours accumulated (default: 0)
</ParamField>

<ParamField body="current_cycles" type="integer">
  Current cycles accumulated (default: 0)
</ParamField>

<ParamField body="last_major_maintenance_date" type="date">
  Date of last major maintenance (YYYY-MM-DD)
</ParamField>

<ParamField body="total_lifetime_hours" type="number">
  Total lifetime hours (default: 0)
</ParamField>

<ParamField body="total_lifetime_cycles" type="integer">
  Total lifetime cycles (default: 0)
</ParamField>

<ParamField body="status" type="enum">
  Aircraft status: `ACTIVE`, `INACTIVE`, `ELIMINATED`, `APPROACHING_MAINTENANCE`, `MAINTENANCE`, `GROUNDED`, `OUT_OF_SERVICE`
</ParamField>

## List Aircraft

<Note>
  Requires authentication with `requireDespachoRead` permission
</Note>

Get a paginated list of aircraft with optional filters.

<CodeGroup>
```bash cURL
curl -X GET "https://api.sigsaho.com/aircrafts?page=1&limit=10&status=ACTIVE&search=Bell" \
  -H "Cookie: token=YOUR_JWT_TOKEN"
```
</CodeGroup>

### Query Parameters

<ParamField query="page" type="integer" default="1">
  Page number (minimum: 1)
</ParamField>

<ParamField query="limit" type="integer" default="10">
  Records per page (minimum: 1, maximum: 100)
</ParamField>

<ParamField query="status" type="enum">
  Filter by status: `ACTIVE`, `INACTIVE`, `ELIMINATED`, `APPROACHING_MAINTENANCE`, `MAINTENANCE`, `GROUNDED`, `OUT_OF_SERVICE`
</ParamField>

<ParamField query="search" type="string">
  Search by name or registration
</ParamField>

### Response

<ResponseField name="data" type="array">
  Array of aircraft objects
</ResponseField>

<ResponseField name="pagination" type="object">
  <Expandable title="Pagination Object">
    <ResponseField name="page" type="integer">
      Current page
    </ResponseField>
    
    <ResponseField name="limit" type="integer">
      Records per page
    </ResponseField>
    
    <ResponseField name="total" type="integer">
      Total records

    <ResponseField name="totalPages" type="integer">

      Whether there is a next page
    </ResponseField>
    
    <ResponseField name="hasPreviousPage" type="boolean">
      Whether there is a previous page
    </ResponseField>
  </Expandable>
</ResponseField>

## Update Aircraft

<Note>
  Requires authentication with `requireDespachoWrite` permission
</Note>

Update an existing aircraft's information.

<CodeGroup>
```bash cURL
curl -X PUT https://api.sigsaho.com/aircrafts/1 \
  -H "Cookie: token=YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Bell 407 Updated",
    "description": "Updated description",
    "status": "ACTIVE"
  }'
```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="integer" required>
  Aircraft ID
</ParamField>

### Request Body

All fields are optional. Only provided fields will be updated.

<ParamField body="name" type="string">
  Aircraft name
</ParamField>

<ParamField body="registration" type="string">
  Aircraft registration
</ParamField>

<ParamField body="description" type="string">
  Aircraft description
</ParamField>

<ParamField body="status" type="enum">
  Aircraft status
</ParamField>

<ResponseExample>
```json Response

  "aircraft": {
    "id": 1,
    "name": "Bell 407 Updated",
    "description": "Updated description",
    "status": "ACTIVE"
```

## Update Aircraft Image

<Note>
  Requires authentication with `requireDespachoWrite` permission
</Note>

Update the image of an existing aircraft. The previous image is automatically deleted.

<CodeGroup>
```bash cURL

```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="integer" required>

</ParamField>

### Request

<ParamField body="image" type="file" required>

</ParamField>

## Authentication

All endpoints require authentication via JWT cookie. The API uses role-based access control:

- **Read operations**: Require `requireDespachoRead` permission
- **Write operations**: Require `requireDespachoWrite` permission
