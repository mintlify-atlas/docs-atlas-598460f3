## Create Aircraft Part

<Note>
  Requires authentication with `requireMantenimientoWrite` permission
</Note>

Create a new aircraft part with complete traceability information.

<CodeGroup>
```bash cURL
curl -X POST https://api.sigsaho.com/aircraft-parts \
  -H "Cookie: token=YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "ata_code": "72-21-00",
    "part_num": "407-040-100-101",
    "description": "Main Rotor Blade",
    "serial_number": "MRB-12345",
    "useful_life_hours": 2000,
    "useful_life_cycles": 1000,
    "useful_date": "2028-03-01",
    "installation_date": "2024-01-15",
    "hours_at_installation": 4500,
    "cycles_at_installation": 900,
    "aircraft_id": 1
  }'
```
</CodeGroup>

</ParamField>

</ParamField>

<ParamField body="serial_number" type="string" required>

</ParamField>

</ParamField>

## List Aircraft Parts

<Note>
  Requires authentication with `requireMantenimientoRead` permission
</Note>

Get a paginated list of aircraft parts with optional filters.

<CodeGroup>
```bash cURL
curl -X GET "https://api.sigsaho.com/aircraft-parts?page=1&limit=10&status=ACTIVE&aircraft_id=1" \
  -H "Cookie: token=YOUR_JWT_TOKEN"
```
</CodeGroup>

### Query Parameters

<ParamField query="page" type="integer" default="1">
  Page number (minimum: 1)
</ParamField>

<ParamField query="limit" type="integer" default="10">
  Records per page (minimum: 1, maximum: 100)
</ParamField>

<ParamField query="status" type="enum">
  Filter by status: `ACTIVE`, `APPROACHING_LIMIT`, `NEAR_LIMIT`, `OVERDUE`, `IN_SERVICE`, `DEPRECATED`
</ParamField>

<ParamField query="aircraft_id" type="integer">
  Filter by aircraft ID
</ParamField>

<ParamField query="search" type="string">
  Search by name, description, serial number, or ATA code
</ParamField>

### Response

<ResponseField name="data" type="array">
  Array of aircraft part objects
</ResponseField>

<ResponseField name="pagination" type="object">

    <ResponseField name="totalPages" type="integer">

    <ResponseField name="hasNextPage" type="boolean">
      Whether there is a next page
    </ResponseField>
    
    <ResponseField name="hasPreviousPage" type="boolean">
      Whether there is a previous page
    </ResponseField>
  </Expandable>
</ResponseField>

## Export Parts Inventory to Excel

<Note>
  Requires authentication with `requireAdminOrDirector` permission (ADMINISTRADOR or DIRECTOR roles only)
</Note>

Export the complete aircraft parts inventory to Excel format (XLSX). The file is streamed directly to the client using chunked transfer encoding without accumulating in server memory.

<CodeGroup>
```bash cURL
curl -X GET "https://api.sigsaho.com/aircraft-parts/export?status=ACTIVE&aircraft_id=1" \
  -H "Cookie: token=YOUR_JWT_TOKEN" \
  -o inventory.xlsx
```
</CodeGroup>

### Query Parameters

<ParamField query="status" type="enum">
  Filter by status: `ACTIVE`, `APPROACHING_LIMIT`, `NEAR_LIMIT`, `OVERDUE`, `IN_SERVICE`, `DEPRECATED`
</ParamField>

<ParamField query="aircraft_id" type="integer">
  Filter by aircraft ID
</ParamField>

<ParamField query="search" type="string">
  Search by description, serial number, or ATA code
</ParamField>

### Response

Returns an Excel file (XLSX) with the parts inventory. The filename format is `inventario-componentes_YYYY-MM-DD.xlsx`.

<ResponseExample>
```text Response Headers
Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
Content-Disposition: attachment; filename="inventario-componentes_2026-03-04.xlsx"
Transfer-Encoding: chunked
```
</ResponseExample>

## Delete Aircraft Part

<Note>
  Requires authentication with `requireMantenimientoWrite` permission
</Note>

Logically delete an aircraft part (soft delete). The status changes to `DEPRECATED`.

<CodeGroup>
```bash cURL
curl -X DELETE "https://api.sigsaho.com/aircraft-parts/1?note=Part replaced during maintenance" \
  -H "Cookie: token=YOUR_JWT_TOKEN"
```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="integer" required>
  Aircraft part ID
</ParamField>

### Query Parameters

<ParamField query="note" type="string">
  Optional note explaining the deletion reason
</ParamField>

<ResponseExample>
```json Response
{
  "message": "Parte de aeronave eliminada exitosamente"
}
```
</ResponseExample>

## Part Status

The system automatically calculates part status based on remaining useful life:

<Card title="ACTIVE" icon="check-circle">
  Part is operating normally with sufficient remaining life
</Card>

<Card title="APPROACHING_LIMIT" icon="exclamation-triangle">
  Warning: Less than 80 hours or 40 cycles remaining, or expiration date within 3 months
</Card>

<Card title="NEAR_LIMIT" icon="exclamation-circle">
  Warning: Less than 20 hours or 10 cycles remaining, or expiration date within 1 month
</Card>

<Card title="OVERDUE" icon="times-circle">
  Critical: Part has exceeded useful life in hours/cycles or expiration date has passed
</Card>

<Card title="IN_SERVICE" icon="wrench">
  Part is currently being serviced
</Card>

<Card title="DEPRECATED" icon="trash">
  Part has been logically deleted (soft delete)
</Card>

## ATA Codes

ATA (Air Transport Association) codes are standardized numbering systems for aircraft systems:

- **72-21-00**: Main Rotor Blades
- **72-30-00**: Tail Rotor
- **71-00-00**: Power Plant
- **53-00-00**: Fuselage
- And many more...

## Authentication

All endpoints require authentication via JWT cookie. The API uses role-based access control:

- **Read operations**: Require `requireMantenimientoRead` permission
- **Write operations**: Require `requireMantenimientoWrite` permission
- **Export operations**: Require `requireAdminOrDirector` permission (ADMINISTRADOR or DIRECTOR roles)
