## Overview

The Dashboard & Analytics module provides comprehensive operational intelligence for SIGSAHO. Monitor fleet performance, track medical mission statistics, identify trends, and make data-driven decisions with real-time visualizations and detailed reports.

<Note>
  All dashboard metrics update in real-time as flight operations, transfers, and maintenance activities are logged in the system.
</Note>

## Statistics Search and Reports

Advanced analytics with custom filtering:

### Search Statistics Page

<Steps>
  <Step title="Access Statistics Search">
    Navigate to **Estadísticas > Búsqueda Estadística** for advanced analytics.
  </Step>
  
  <Step title="Select Date Range">
    Choose analysis period:
    - Custom date range picker

  <Step title="Apply Filters">

    - Status (Finished, Canceled, In Process)
    - Aircraft used
    - Origin/Destination region
    
    **Medical Filters (if applicable):**
    - Diagnosis type
    - Medical transfer category
    - Hospital origin/destination
    - Patient age range
    
    **Crew Filters:**
    - Pilot in command
    - Medical personnel
    - Crew role
  </Step>
  
  <Step title="View Results">
    System displays comprehensive statistics:
    
    **Summary KPIs:**
    - Total transfers matching filters
    - Total flight hours
    - Total patients (if medical)
    - Average mission duration
    - Total fuel consumed
    
    **Visualizations:**
    - Timeline chart of activity
    - Breakdown by type/category
    - Geographic distribution map
    - Top aircraft/pilots
  </Step>
  
  <Step title="Export Report">
    Generate downloadable reports:
    - Excel export with raw data
    - PDF summary report with charts
    - CSV for further analysis
    - Formatted for regulatory submission
  </Step>
</Steps>

|------|----------------|------------|----------------|------------------|
| Administrator | ✓ | ✓ | ✓ | ✓ |
| Operations Manager | ✓ | ✓ | ✓ | ✗ |
| Dispatcher | ✓ | ✓ | ✓ | ✗ |
| Pilot | ✓* | ✓* | ✗ | ✗ |
| Medical Coordinator | ✓* | ✓* | ✓** | ✗ |
| Viewer | ✓ | ✓ | ✗ | ✗ |

*Limited to relevant data (pilots see flight-related, medical coordinators see medical transfers)
**Medical coordinators can only export medical transfer reports

## Data Refresh and Performance

<Card title="Real-Time Updates" icon="sync">
  **Dashboard Data Refresh Strategy**
  
  SIGSAHO uses intelligent caching for optimal performance:
  
  **Auto-Refresh:**
  - Last Flights Widget: Updates every 60 seconds
  - Urgent Alerts Widget: Updates every 30 seconds
  - KPI Cards: Updates every 5 minutes
  - Charts: Updates on page load and manual refresh
  
  **Manual Refresh:**
  - Click refresh icon to force update
  - Useful after logging new operations

  - React Query handles caching automatically

## Troubleshooting

<Accordion title="Dashboard Shows No Data">
  **Possible Causes:**
  - No flight operations logged yet
  - Date range filter excludes all data
  - User permissions restrict data visibility
  
  **Solution:**
  1. Verify date range includes actual operations
  2. Remove restrictive filters
  3. Check user role and permissions
  4. Manually refresh the dashboard
</Accordion>

<Accordion title="Charts Not Loading">
  **Possible Causes:**
  - Network connectivity issue
  - Backend API error
  - Browser compatibility issue
  
  **Solution:**
  1. Check network connection
  2. Refresh the page
  3. Clear browser cache
  4. Try a different browser (Chrome, Firefox, Edge)
</Accordion>

<Accordion title="Statistics Seem Incorrect">
  **Possible Causes:**
  - Data not yet synced from recent operations
  - Cache not invalidated
  - Incomplete transfer records (status still "In Process")
  
  **Solution:**
  1. Wait a few minutes for data to sync
  2. Manually refresh dashboard
  3. Verify recent transfers have status "Finished"
  4. Check for incomplete records and finalize them
</Accordion>

## Alert Configuration

### Configuring Alert Thresholds

Administrators can customize when alerts trigger:

<Steps>
  <Step title="Access Alert Settings">
    Navigate to **Seguimiento > Configuración de Alertas**.
  </Step>
  
  <Step title="Configure Document Expiration Warnings">
    Set number of days before expiration to trigger alerts:
    - User documents (licenses, passports, medicals): Default 30 days
    - Aircraft documents (registration, insurance): Default 60 days
    - Training certifications: Default 30 days
  </Step>
  
  <Step title="Configure Maintenance Thresholds">
    Set percentage thresholds for maintenance alerts:
    - Warning threshold: Default 70% (amber alert)
    - Critical threshold: Default 85% (red alert)
    - Overdue: 100%+ (critical red alert)
  </Step>
  
  <Step title="Configure Training Alerts">
    Set training expiration warnings:
    - Initial warning: 60 days before expiration
    - Urgent warning: 30 days before expiration
    - Overdue: Past expiration date
  </Step>
  
  <Step title="Save Configuration">
    Click **"Guardar Configuración"** to apply new thresholds. Changes take effect immediately.
  </Step>
</Steps>
