## Get up and running

This guide walks you through setting up SIGSAHO locally, logging in for the first time, and navigating the dashboard.

<Steps>

### Clone the repository

Clone the SIGSAHO repository to your local machine:

```bash
git clone <repository-url>
cd sigsaho
```

### Set up the backend

Navigate to the backend directory and configure your environment:

```bash
cd back
cp .env.example .env
```

Configure your `.env` file with the following required variables:

```bash .env
# Database Configuration
POSTGRES_DB=estadistica-hangar
DB_PORT=5432
DB_HOST=localhost
POSTGRES_USER=postgres
POSTGRES_PASSWORD=your_password

# Server Configuration
PORT=3001
API_DB_SYNC=true
API_IS_PROD=false

# Authentication
TOKEN_SECRET=your-secret-token-here
SECRET_PASSWORD_ENCRYPT=your-password-encryption-key
JWT_ACCESS_TOKEN_EXPIRY=15m
JWT_REFRESH_TOKEN_EXPIRY_DAYS=7

# CORS Configuration
CORS_ORIGIN=*

# File Upload Configuration
NODE_ENV=development
UPLOAD_MAX_SIZE=10485760
UPLOADS_DIR=uploads
```

<Warning>
  Update `TOKEN_SECRET` and `SECRET_PASSWORD_ENCRYPT` with secure random values in production environments.
</Warning>

Install dependencies and start the backend server:

```bash
npm install
npm run dev
```

The API will be available at `http://localhost:3001`.

### Set up the frontend

In a new terminal, navigate to the frontend directory and install dependencies:

```bash
cd ../frontend
npm install
npm run dev
```

The frontend will be available at `http://localhost:5173`.

### Log in to SIGSAHO

Navigate to `http://localhost:5173` in your browser. You'll see the login screen.

<Frame>
  <img src="/images/login-screen.png" alt="SIGSAHO Login Screen" />
</Frame>

Enter your credentials:

- **Username or Email**: Your assigned username or email address
- **Password**: Your account password
- **Remember me** (optional): Keep your session active for 7 days

```typescript
// The login endpoint accepts either username or email
POST /auth/login
{
  "identifier": "username or email@hangar.com",
  "password": "your-password"
}
```

<Tip>
  The system supports authentication with either username or email. Enter whichever you prefer in the identifier field.
</Tip>

### Explore the dashboard

After successful login, you'll be redirected to the main dashboard. The system uses secure httpOnly cookies for authentication.

#### Authentication flow

SIGSAHO implements a dual-token authentication system:

1. **Access Token**: Short-lived (15 minutes), used for API requests
2. **Refresh Token**: Long-lived (7 days), used to renew access tokens

```typescript
// Response format from auth.controller.ts:40-53
{
  "message": "Login exitoso",
  "user": {
    "id": 1,
    "name": "John",
    "last_name": "Doe",
    "second_last_name": "Smith",
    "username": "jdoe",
    "email": "jdoe@hangar.com",
    "roles": [...],
    "login_access": true,
    "status": "ACTIVE"
  }
}
```

#### User model

The user entity contains the following fields:

```typescript user.entity.ts:40-86
@Entity('users')
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  name: string;

  @Column()
  last_name: string;

  @Column()
  second_last_name: string;

  @Column({ type: 'varchar', nullable: true })
  @Index({ unique: true })
  username: string | null;

  @Column()
  @Index({ unique: true })
  email: string;

  @Column({ nullable: true })
  phone: string;

  @Column()
  login_access: boolean;

  @Column()
  weight: number;

  @Column({ nullable: true })
  license_number: string;

  @Column({ nullable: true })
  birth_date: Date;

  @Column({ nullable: true })
  position: string;

  @Column({ type: 'enum', enum: UserStatus, default: UserStatus.ACTIVE })
  status: UserStatus;

  @ManyToMany(() => Roles)
  roles: Roles[];
}
```

## Authentication API reference

### Available endpoints

<AccordionGroup>
  <Accordion title="POST /auth/login" icon="right-to-bracket">
    Authenticate a user with username/email and password.
    
    **Rate Limit**: 5 attempts per 15 minutes per IP
    
    **Request Body**:
    ```json
    {
      "identifier": "username or email",
      "password": "your-password"
    }
    ```
    
    **Response**: Returns user object and sets httpOnly cookies
  </Accordion>

  <Accordion title="POST /auth/refresh" icon="arrows-rotate">
    Renew the access token using the refresh token from cookies.
    
    **Rate Limit**: 10 requests per 15 minutes per IP
    
    **Response**: Updates accessToken cookie with new token
  </Accordion>

  <Accordion title="POST /auth/logout" icon="right-from-bracket">
    Invalidate the current session and clear authentication cookies.
    
    **Rate Limit**: 20 requests per 15 minutes per IP
    
    **Response**: Clears all authentication cookies
  </Accordion>

  <Accordion title="GET /auth/me" icon="user">
    Get the current authenticated user's information.
    
    **Requires**: Valid access token in httpOnly cookie
    
    **Response**: Returns full user object with roles
  </Accordion>
</AccordionGroup>
